## 7-Phase Workflow

Follow the phases strictly. Use `references/execution-protocol.md` for detailed instructions at each phase.

### Phase 1: Phenomenon Mining

Build a library of real-world systems exhibiting the target behavior. Select from the domain template's Pre-Filled Phenomenon Library or add your own.

**Required:** 5-10 phenomena from at least 3 distinct domains.

**Phenomenon Library:**

| Name | Domain | Observed Behavior | Why It Matters |
|------|--------|-------------------|----------------|
| | | | |
| | | | |

### Phase 2: Tension Mining

Identify forces that shape the phenomena. Select from the domain template's Pre-Filled Tension Checklist or add your own.

**Required:** At least 5 tensions with clear Force A/B.

**Tension Map:**

| Tension ID | Force A | Force B | Why Ineliminable | Over-optimize A | Over-optimize B |
|------------|---------|---------|------------------|-----------------|-----------------|
| | | | | | |
| | | | | | |

### Phase 3: Invariant Mining

Extract principles that appear across multiple tensions and domains. Select from the domain template's Pre-Filled Invariant Checklist or add your own.

**Required:** At least 3 invariants, each with supporting phenomena from distinct domains.

**Invariant Library:**

| Invariant ID | Statement | Supporting Domains | Explanation | Boundary Conditions |
|--------------|-----------|-------------------|-------------|---------------------|
| | | | | |
| | | | | |

### Phase 4: Mechanism Mining

Study how reality resolves the tensions. Select from the domain template's Pre-Filled Mechanism Library or add your own.

**Required:** At least 3 mechanisms, each mapped to at least one tension.

**Mechanism Library:**

| Mechanism | Function | Related Tensions | Related Invariants | Failure Modes |
|-----------|----------|------------------|--------------------|---------------|
| | | | | |
| | | | | |

### Phase 5: System Synthesis

Combine tensions, invariants, and mechanisms into a coherent system model.

**System Model:**

- **Core Tensions:** (List primary tensions)
- **Core Invariants:** (List primary invariants)
- **Core Mechanisms:** (List essential mechanisms)
- **Expected Behaviors:** (What is predicted?)
- **Failure Modes:** (What breaks and under what conditions?)

### Phase 6: Algorithm Synthesis

Derive algorithm candidates from the mechanisms.

**Algorithm Candidates:**

| Name | Core Idea | Formal Description | Advantages | Limitations |
|------|-----------|-------------------|------------|-------------|
| | | | | |
| | | | | |

### Phase 7: Destruction Phase

Attack the model. Identify weak assumptions, missing tensions, missing invariants, missing mechanisms, and propose redesigns.

**Failure Analysis:**

| Category | Findings |
|----------|----------|
| **Weak Assumptions** | (At least 3) |
| **Missing Tensions** | (At least 2) |
| **Missing Invariants** | (At least 1) |
| **Missing Mechanisms** | (At least 2) |
| **Possible Redesigns** | (At least 2 concrete proposals) |
