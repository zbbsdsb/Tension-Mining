> **大多数人寻找解决方案。杰出的研究者寻找张力。**

Tension Mining（张力挖掘）是一种研究方法论，用于发现隐藏在复杂系统内部的不变量（Invariant）。它以 **AI 可执行的 Skill** 形式工作，兼容 Claude Code、TRAE、Cursor、Windsurf 以及任何支持 Markdown 格式 Skill 文件的工具。

## 为什么需要张力挖掘？

大多数创新工作流程开始得太晚了。

它们从算法、架构、实现、优化开始。但最具影响力的系统很少诞生于优化。

- **PageRank** 始于一个关于"重要性"的问题 —— 而不是矩阵方程。
- **Bitcoin** 始于去中心化与信任之间的张力 —— 而不是区块链数据结构。
- **Wikipedia** 始于开放性与可靠性之间的张力 —— 而不是版本控制系统。
- **Transformer** 始于一个关于循环是否必要的问题 —— 而不是注意力机制。

突破出现在算法之前很久。它出现在隐藏的张力终于被看见的那一刻。

**Tension Mining 是一副透镜。** 它的目的很简单：帮助研究者在尝试设计系统之前，先发现塑造系统的力量。

### 核心思想

每一个持久存在的系统，都被一组未解决的张力所塑造。

| 领域 | 张力 |
|------|------|
| 组织 | 自由 ↔ 效率 |
| 社会 | 秩序 ↔ 创新 |
| AI 智能体 | 自主 ↔ 控制 |
| NPC 世界 | 生存 ↔ 探索 |
| 产品 | 简洁 ↔ 能力 |
| 市场 | 竞争 ↔ 合作 |

大多数人关注行为。Tension Mining 关注行为之下的力量。

## 学术基础

Tension Mining 汲取并综合了多个既有研究传统中的概念。

| 领域 | 核心参考文献 | 关系 |
|------|------------|------|
| 复杂系统 | Mitchell (2009) [1], Holland (1995) [2] | 涌现、自组织和适应的核心框架 |
| 设计思维 | Schön (1983) [3], Cross (2006) [4] | 现象优先的方法，在方案之前重新定义问题 |
| 系统思维 | Meadows (2008) [5], Senge (1990) [6] | 反馈循环、杠杆点、系统结构分析 |
| 赛尼芬框架 | Snowden & Boone (2007) [7] | 区分有序问题域与复杂问题域 |
| 研究方法论 | Kuhn (1962) [8], Lakatos (1970) [9] | 范式转移、研究纲领结构、可证伪性 |
| 进化理论 | Dawkins (1976) [10], Boyd & Richerson (1985) [11] | 变异-选择-保留作为不变量发现引擎 |
| 网络科学 | Barabási & Albert (1999) [12], Watts & Strogatz (1998) [13] | 局部交互产生的涌现特性、小世界现象 |

这些基础并非装饰。每一个都直接指导管线的特定阶段：
- **复杂系统** → 阶段 1（现象必须跨越领域，而非强化领域短视）
- **设计思维** → 阶段 2-3（先有张力后有方案，重新定义问题为核心技能）
- **系统思维** → 阶段 4-5（机制通过反馈交互，而非线性因果）
- **赛尼芬框架** → 阶段 7（破坏揭示问题是处于繁杂领域还是复杂领域）
- **进化理论** → 阶段 3（不变量通过跨领域存续被选择，而非设计者意图）

请参阅[方法论入门](./references/methodology-primer.md)以了解与相关方法论的详细比较及完整阅读清单。

## 快速开始

### 1. 安装

将本仓库克隆到你的 AI 工具的 Skill 目录：

```bash
# Claude Code — 项目级（推荐）
git clone https://github.com/zbbsdsb/Tension-Mining.git .claude/skills/tension-mining

# Claude Code — 用户级（所有项目）
git clone https://github.com/zbbsdsb/Tension-Mining.git ~/.claude/skills/tension-mining

# TRAE
git clone https://github.com/zbbsdsb/Tension-Mining.git .trae/skills/tension-mining

# Cursor / Windsurf / 其他
# 将仓库放在任何可访问的位置。在你的项目指令中引用 SKILL.md。
```

### 2. 使用

**自动触发** —— 用自然语言描述一个复杂系统问题：

> "我想为一个 P2P 市场设计去中心化身份系统。"

AI 检测到触发条件后自动激活 Tension Mining。

**手动触发** —— 直接调用 Skill：

| 工具 | 命令 |
|------|------|
| Claude Code | `/tension-mining` |
| TRAE | AI 基于 SKILL.md 描述自动激活 |
| Cursor / Windsurf | 在你的 `.cursorrules` 或项目指令中引用 `SKILL.md` |

### 3. 跟随七个阶段

AI 会引导你经历 7 个阶段，每次只问一个问题：

```
1. 现象挖掘  →  从 3+ 领域收集 5-10 个真实案例
2. 张力挖掘  →  识别 5+ 个不可消除的权衡
3. 不变量挖掘 →  提取 3+ 个跨领域原理
4. 机制挖掘  →  研究现实如何解决这些张力
5. 系统综合  →  整合为一个连贯的模型
6. 算法综合  →  从机制推导算法（只在此时）
7. 摧毁阶段  →  攻击你自己的模型
```

**核心原则：** 不要从解决方案开始。从现实开始。

## 工作原理

### 设计模式

Tension Mining 采用 **管道 + 反转 + 生成器** 混合模式：

- **管道（Pipeline）** —— 7 个阶段严格按顺序执行。每个阶段有门控条件；不满足就不能进入下一阶段。
- **反转（Inversion）** —— AI 分阶段采访你，每次只问一个问题。你驱动内容；AI 强制执行方法论。
- **生成器（Generator）** —— 输出遵循结构化模板，确保完整性。

### 防作弊机制

- **常见合理化借口表** —— 对抗 AI 跳过步骤的倾向（"我已经知道张力了，直接跳到算法吧"）
- **红旗信号** —— 可观察的行为，表明方法论正在被违反
- **质量评分表** —— 五维自评（D1-D5，0-15 分），附加到每个输出

### 渐进式披露

`SKILL.md` 是一个简洁的激活骨架（约 80 行）。详细的阶段指令、图谱和模板按需加载，保持初始上下文最小化。

## 七个阶段

### 1. 现象挖掘（Phenomenon Mining）

> 哪些系统已经表现出这种行为？

在建立抽象之前先观察现实。从蚁群、公司、城市、生态系统、在线社区、开源项目中构建现象库。

### 2. 张力挖掘（Tension Mining）

> 什么权衡永远无法完全消除？

识别将系统拉向不同方向的力量。构建张力地图。

### 3. 不变量挖掘（Invariant Mining）

> 什么规律不受领域限制始终成立？

在不相关的系统中寻找重复出现的模式。提取不变量。

### 4. 机制挖掘（Mechanism Mining）

> 自然、社会或技术中已经存在什么机制？

研究现实如何解决张力。构建机制库。

### 5. 系统综合（System Synthesis）

> 能解释这个系统的最小模型是什么？

将张力、不变量和机制整合为一个连贯的模型。识别哪些张力是主要的，哪些机制是本质的，以及存在什么失效模式。

### 6. 算法综合（Algorithm Synthesis）

> 如果这个机制是真实的，应该如何实现？

只在此时设计算法。让它们从机制中自然涌现。

### 7. 摧毁阶段（Destruction Phase）

> 什么假设会失效？什么边界情况会打破系统？

攻击模型。假设它是错的。识别薄弱假设、缺失张力和可能的重新设计。

## 案例研究

每个案例都完整走过 7 阶段管道：

| 案例 | 领域 | 关键张力 | 关键不变量 |
|------|------|---------|-----------|
| [PageRank](./examples/page-rank.md) | 信息检索 | 局部 vs 全局，自由 vs 效率 | 局部规则创造全局秩序，梯度驱动运动 |
| [Transformer](./examples/transformer.md) | 深度学习 | 同步 vs 异步，压缩 vs 保真 | 压缩揭示结构，梯度驱动运动 |
| [Bitcoin](./examples/bitcoin.md) | 加密货币 | 中心化 vs 去中心化，竞争 vs 合作 | 身份驱动合作，权衡不可逃避 |
| [Git](./examples/git.md) | 版本控制 | 一致性 vs 可用性，局部 vs 全局 | 局部规则创造全局秩序，身份驱动合作 |
| [Wikipedia](./examples/wikipedia.md) | 协作知识 | 自由 vs 效率，个体 vs 集体 | 身份驱动合作，压缩揭示结构 |
| [NPC Society](./examples/npc-society.md) | 多智能体系统 | 生存 vs 探索，个体 vs 集体 | 局部规则创造全局秩序，变异促成选择 |
| [Agent Organization](./examples/agent-organization.md) | AI 协调 | 自主 vs 控制，中心化 vs 去中心化 | 身份驱动合作，反馈循环稳定 |
| [对话演示](./examples/dialogue-example.md) | 去中心化身份 | *完整 7 阶段交互演示* | *推荐首次阅读* |
| [共识协议](./examples/consensus-protocols.md) | 分布式系统 | 安全性 vs 活性，一致性 vs 可用性 | 局部规则创造全局秩序，权衡不可逃避 |
| [蚁群觅食](./examples/ant-colony.md) | 生物系统 | 个体 vs 集体，生存 vs 探索 | 局部规则创造全局秩序，变异促成选择 |
| [市场效率](./examples/market-efficiency.md) | 经济学 | 秩序 vs 创新，竞争 vs 合作 | 模式放大力量，权衡不可逃避 |

## 这不是什么

- **不是** 提示词集合
- **不是** 头脑风暴模板
- **不是** 生产力框架
- **不是** 通往创新的保证路径
- **不是** 视觉设计系统

Tension Mining 是一副透镜。它的目的很简单：帮助研究者在尝试设计系统之前，先发现塑造系统的力量。
