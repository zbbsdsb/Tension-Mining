# Tension Mining — Quality Rubric

> Load this file after completing the destruction phase (Phase 7).
> Self-evaluate the analysis against these criteria before presenting the final output.

---

## D1 — Phenomenon Quality (0-3)

| Score | Criteria |
|-------|----------|
| 0 | No phenomena identified, or only 1 domain represented |
| 1 | 3-5 phenomena from 2 domains |
| 2 | 5-10 phenomena from 3+ domains |
| 3 | 10+ phenomena from 4+ domains, including at least one surprising cross-domain analogy |

## D2 — Tension Depth (0-3)

| Score | Criteria |
|-------|----------|
| 0 | Fewer than 3 tensions, or tensions are merely "pros vs cons" |
| 1 | 3-4 tensions listed with Force A/B clearly distinguished |
| 2 | 5+ tensions with cross-domain examples and over-optimization consequences |
| 3 | Tensions include "why ineliminable" reasoning and clear mapping to invariants |

## D3 — Invariant Validity (0-3)

| Score | Criteria |
|-------|----------|
| 0 | No invariants, or invariants are domain-specific (not cross-domain) |
| 1 | 1-2 invariants with supporting phenomena from 2 domains |
| 2 | 3+ invariants each with 3+ supporting phenomena from distinct domains |
| 3 | Invariants include boundary conditions and falsifiable claims |

## D4 — Mechanism Map (0-3)

| Score | Criteria |
|-------|----------|
| 0 | No mechanisms identified |
| 1 | Mechanisms listed but not mapped to tensions or invariants |
| 2 | Each mechanism maps to at least one tension and one invariant |
| 3 | Mechanisms include function explanation and failure analysis |

## D5 — Destruction Depth (0-3)

| Score | Criteria |
|-------|----------|
| 0 | No destruction phase, or only surface-level criticism |
| 1 | Identifies weak assumptions |
| 2 | Identifies missing tensions, invariants, and mechanisms |
| 3 | Proposes concrete redesigns based on failure analysis |

## Overall Assessment

| Total Score | Rating | Action |
|-------------|--------|--------|
| 12-15 | Professional-grade | Analysis ready for publication or implementation |
| 8-11 | Solid analysis | Consider deeper cross-domain research before finalizing |
| 4-7 | Partial analysis | Revisit phases with low scores and iterate |
| 0-3 | Incomplete | Restart from Phase 1 with a broader scope |

**Note:** Append the rubric scores table to the analysis output. The scores are part of the deliverable.
