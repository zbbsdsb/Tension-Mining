# Support

Welcome to the Tension Mining community! There are several ways to get help,
share your work, and connect with other users.

## Documentation

Before reaching out, check the
[Methodology Primer](./references/methodology-primer.md) — it includes a
comprehensive FAQ section covering common questions about the methodology,
its relationship to other frameworks, and practical usage tips.

## Support Channels

### GitHub Issues (Bug Reports & Feature Requests)

Use GitHub Issues for concrete, actionable items:

- **Bug reports**: Something is broken or not working as expected. Open an
  issue using the [Bug Report](.github/ISSUE_TEMPLATE/bug-report.yml)
  template.
- **Feature requests**: You have an idea for improving the methodology, the
  tooling, or the documentation. Open an issue using the
  [Feature Request](.github/ISSUE_TEMPLATE/feature-request.yml) template.

Before opening a new issue, please search the existing issues to avoid
duplicates.

### GitHub Discussions (Q&A, Show and Tell, General Chat)

Use GitHub Discussions for open-ended conversations:

- **Q&A**: Ask questions about applying the methodology, understanding
  tensions and invariants, or using the skill in different AI tools.
- **Show and Tell**: Share your case studies, tension maps, invariant
  discoveries, or examples of Tension Mining in action. This is the best
  place to get feedback from the community.
- **General**: Discuss anything related to complex systems, cross-domain
  invariants, or methodology design.

Join the conversation at
https://github.com/zbbsdsb/Tension-Mining/discussions.

### CONTRIBUTING.md

If you would like to contribute a new tension, invariant, case study, or
template, please read the
[Contributing Guide](./CONTRIBUTING.md). It covers:

- How to add tensions and invariants to the atlas
- How to write and submit case studies
- The [CORE]/[EXPERIMENTAL] promotion criteria
- The PR checklist and validation process
- Academic credit and co-authorship opportunities
